# 마이프로덕트 (MyProduct) — VISION

> **이 문서의 위치**
> 이 문서는 마이프로덕트의 **마스터 비전**입니다. 모든 미래 의사결정의 기준점이 됩니다.
> 기존 `PRD.md` (2026-04-23 작성, Phase 1 발견·피드백·증명서 플랫폼)는 이 비전의 첫 단계입니다.

> **버전**: 1.0
> **작성일**: 2026-04-29
> **다음 검토일**: Phase 1 출시 + 1개월 후

---

## 1. 한 줄 비전

> **"제품을 만든 메이커가 그것을 사업으로 키우기까지의 모든 단계를, 한 곳에서."**

마이프로덕트는 단순한 발견 플랫폼이 아닙니다. 메이커의 **라이프사이클 운영 OS**입니다.

---

## 2. 우리가 푸는 문제

AI와 바이브코딩이 보편화되면서, 한국에서 사이드프로젝트를 만드는 사람은 폭발적으로 늘고 있습니다. 그러나 그들이 **만든 다음**에 부딪히는 문제는 누구도 풀어주지 않습니다.

| 단계 | 메이커가 부딪히는 벽 | 현재 한국 시장 |
|------|-------------------|----------------|
| 만든 직후 | "이거 어디 보여주지" | 디스퀴엇 (트래픽 약함) |
| 발견 후 | "혼자 키우긴 벅찬데" | 비긴메이트 (아이디어 단계 한정) |
| 사업화 직전 | "정부 지원사업 받고 싶다" | 기업마당 (UX 형편없음) |
| 신청 시 | "내 제품의 시장 검증 데이터를 어떻게 증명하지" | 없음 (모두가 자기 설문 100명으로 떼움) |
| 운영 한계 | "지쳤는데 사겠다는 사람은 있을 텐데" | 없음 (Acquire.com 한국판 부재) |

**한국 메이커의 디폴트 결말**: "사이드프로젝트 만들고 → 발견 안 됨 → 동료 못 구함 → 운영 지침 없음 → 방치 → 죽음."

이 깔때기를 끊어주는 곳이 마이프로덕트입니다.

---

## 3. 누구를 위한 서비스인가

### 핵심 타깃

**바이브코딩·노코드·정식 개발 모두 포함하여, 한국에서 사이드프로젝트나 1인 제품을 만든 메이커.**

구체 페르소나:
- 본업이 있는 직장인 사이드 프로젝트 메이커
- AI 시대 들어 처음 제품을 만들어본 비개발자
- 1~2개 작품 만들어봤지만 사업으로 못 키운 인디 메이커
- 정부 창업 지원사업에 도전하려는 예비창업가

### 명시적 제외

- 이미 시리즈 A 이상 받은 스타트업 (다른 SaaS가 더 잘 맞음)
- 아이디어 단계만 있고 만들어본 게 없는 사람 (비긴메이트가 더 맞음)
- B2B 엔터프라이즈 SW 개발자 (관심사 다름)

---

## 4. 핵심 차별점 — 왜 마이프로덕트만 가능한가

마이프로덕트의 진짜 해자는 **데이터 흐름**입니다.

```
F1 (제품 등록)
  ↓ 자동
F3 (트랙션 자동 측정)
  ↓ 자동
F7 (트랙션 리포트 — 객관적 시장 검증 PDF)
  ↓ 자동 첨부
F9 (정부 지원사업 신청 시 결정적 무기)
```

**다른 사업계획서 도구는 메이커가 본인 입으로 트랙션을 입력해야 합니다.** 그 숫자는 부풀려졌을 가능성이 있어 심사위원이 신뢰하지 않습니다.

마이프로덕트는 **자기 플랫폼에서 직접 측정한 데이터**를 발급합니다. 메이커가 부풀릴 수 없습니다. 정부 지원사업 심사위원이 가장 보고 싶어 하는 객관적 외부 검증 데이터가 자동으로 나옵니다.

이것이 **카피 불가능한 해자**입니다. 어떤 SaaS도 마이프로덕트의 트랙션 데이터를 갖고 있지 않습니다.

## 5. 9개 기능 정의

기능 번호는 의미만 갖습니다 (개발 순서가 아닙니다).

### F1. 제품 등록 (무료)
메이커가 자기 사이드프로젝트를 등록한다. 제품명, 한 줄 소개, URL, 스크린샷, 카테고리, 기술스택, 메이커 정보. 모든 다음 기능의 인풋이 된다.

### F2. 피드백 받기 (무료)
다른 사용자가 좋아요·댓글·구조화 피드백(UX/아이디어/시장성/기술 4축)을 남긴다.

### F3. 트랙션 자동 측정 (무료, 백그라운드)
조회수·좋아요·댓글·외부 클릭·카테고리 순위를 자동 누적한다. 메이커가 손 댈 필요 없음.

### F4. 메이커 프로필 = 포트폴리오 (무료)
한 메이커가 등록한 모든 제품이 한 프로필에 자동 누적된다. 자체 SEO 페이지가 된다. "구직/협업 의향" 토글 포함.

### F5. 동료 찾기 (기본 무료, 프리미엄 노출 9,900원)
"동료 찾는 중" 토글로 합류자 모집. 비슷한 단계 메이커 간 매칭.
**핵심 차별점**: 비긴메이트는 "아이디어 단계", 우리는 "이미 만든 제품에 합류". 합류자 입장에서 검증된 베팅.

### F7. 트랙션 리포트 생성 (1건 9,900원 / 무제한 구독 19,900원)
F3에서 자동 누적된 데이터를 사업계획서·IR 부록 형식의 1~2장짜리 PDF로 변환. 그래프 + 카테고리 순위 + 사용자 반응 인용. 한글/영문 선택.

### F9. 정부 지원사업 매칭 — "받자" (기본 무료, 무제한 매칭+알림 월 9,900원)
매일 크롤링되는 정부 지원사업 공고 DB(중앙부처+지자체 500+개)를 메이커 프로필·제품 카테고리와 매칭. 매주 다이제스트, 마감 알림(D-7/3/1), 결과 추적, 차선 안내.

### F11. 매각/거래 마켓 (장기, 거래액 5~10% 수수료)
운영 지친 메이커의 매각 의향 등록 → 인수자 매칭 → 에스크로 안내. 마이크로 인수 마켓플레이스.

### F12. Affiliate 추천 (메이커 무료, 외부 수수료)
사업자등록·PG·세무·비상주사무실·CRM 등 외부 서비스를 메이커 단계에 맞춰 추천하고 affiliate 수수료 수령.

### F13. B2B 라이선스 (장기, 월 30~100만원)
액셀러레이터·창업지원기관에 자기 보육 기업 단위 라이선스 판매. 분기별 트렌드 리포트 별도.

### 명시적으로 안 하는 것
- **F6 약관/법무 문서 자동 생성** — 통합 가치 없음, GPT로 충분, 신뢰 검증 어려움
- **F8 사업계획서/IR 덱 자동 작성** — 레드오션, 양식 다양성 대응 무리, F7만으로 차별점 충분
- **F10 합격 후 정산 도구** — 모집단 너무 작음, 양식 다양성 대응 부담, 삼쩜삼·자비스 영역

(F6·F8·F10 번호는 결번 처리. F1~F5, F7, F9, F11~F13.)

## 6. 메이커 라이프사이클 (6단계)

```
1. 만들기 — 마이프로덕트가 안 함 (AI 도구들 영역)
   ↓
2. 발견·피드백 ← F1, F2, F3, F4
   ↓
3. 동료·파트너 ← F5
   ↓
4. 사업화 준비 ← F7 (트랙션 리포트)
   ↓
5. 외부 자원 ← F9 (정부 지원사업), F12 (affiliate)
   ↓
6. 운영·확장·엑싯 ← F11, F13
```

**한 메이커가 1년에 걸쳐 이 단계를 통과하면서 우리에게 여러 번 결제한다.** 단일 기능 결제가 아니라 라이프사이클 ARPU.

## 9. 핵심 가설 & 검증 방법

비전이 옳은지 확인하려면 다음 가설들이 맞아야 한다.

### 가설 1: 한국 메이커는 "발견" 다음 단계를 원한다
**검증**: Phase 1 출시 후 등록자에게 5문항 설문. "다음에 가장 도와줬으면 하는 단계"
**실패 시**: F2·F4까지만 풀고 사업 모델 재검토

### 가설 2: 트랙션 리포트는 정부 지원사업 신청자에게 진짜 가치다
**검증**: Phase 2에서 F7 출시 후, 사업계획서에 첨부한 사용자가 합격률에서 유의미하게 우세한지 확인
**실패 시**: F7을 IR/투자자용으로 재포지셔닝

### 가설 3: 메이커 본인이 월 9,900원을 낸다
**검증**: F9 출시 후 3개월 결제 전환율. 무료 → 유료 전환 5% 이상이면 성공
**실패 시**: F9를 무료로 풀고 F12(Affiliate)·F13(B2B)으로 수익 이전

### 가설 4: B2B 영업 가능
**검증**: Phase 3 후반에 액셀러레이터 5곳 콜드 미팅. 1곳이라도 파일럿 합의하면 진행
**실패 시**: B2C 구독 + Affiliate에 집중

## 10. 의사결정 원칙

새 기능 추가나 변경 시 다음 질문을 통과해야 한다.

1. **이게 기존 마이프로덕트 데이터를 활용하는가?**
   활용하지 않으면 다른 SaaS와 차별점이 없다. 추가 보류.

2. **이게 메이커 라이프사이클의 어느 단계에 속하는가?**
   소속 단계가 명확하지 않으면 비전과 어긋난다.

3. **솔로 운영자가 유지보수 가능한가?**
   양식 무한 대응·매년 정책 변경 추적 등은 즉시 거부.

4. **데이터를 흘리는가, 가둬두는가?**
   기능이 마이프로덕트 안에서 다음 단계로 데이터를 흘려야 통합 가치가 있다.

5. **사용자가 이걸 안 쓰면 무슨 일이 일어나는가?**
   "아무 일도 일어나지 않는다"면 안 만든다.

## 11. 명시적으로 안 하는 것

- **AI 사업계획서 도구** (F8 삭제) — 레드오션, 차별점 약함
- **AI 약관/법무 문서 도구** (F6 삭제) — GPT 무료 대체재
- **합격 후 정산 도구** (F10 삭제) — 시장 작고 양식 무한
- **만드는 단계 도구** — Cursor·Claude Code 영역, 우리 안 들어감
- **B2B 엔터프라이즈** — 자원·도메인 부족
- **글로벌** — 1년차에는 한국에 집중. 다국어 지원도 1년 후

## 14. 운영자가 흔들릴 때 보는 메모

> 마이프로덕트가 죽 쒀 보일 때, 트래픽 안 올 때, 메이커들이 결제 안 할 때, 다음 두 가지를 기억하자.
>
> 1. **이건 디스퀴엇이 아니다.** 발견 플랫폼은 입구일 뿐, 진짜 가치는 라이프사이클 통합에 있다. Phase 1만 보고 판단하지 말 것.
>
> 2. **해자는 데이터다.** 트랙션 자동 측정과 그 데이터를 신청서로 흘리는 라인. 이게 작동하기 시작하면 카피 불가능하다. 작동할 때까지 버틴다.
